# 🚀 Kozi Tábor - Project & Deployment Tool

This repository contains the source code for the **Kozitábor** web application and its dedicated developer utility, the **DevTool**. The system is designed for rapid development, automated builds, and secure, "one-click" deployments.

---

## 🛠️ Local Environment Setup

To run the project and the DevTool locally, ensure you have the following tools installed:

### 1. Node.js & npm
The engine behind the frontend and the DevTool.
* **Version:** v18.x or newer recommended.
* **Link:** [Download Node.js](https://nodejs.org/)

### 2. Docker & Docker Desktop
Used for containerization and building reproducible images.
* **Required:** Docker Engine and Docker Compose.
* **Link:** [Install Docker Desktop](https://www.docker.com/get-started)

### 3. Vite
The build tool for lightning-fast frontend development.
* **Link:** [Vite Documentation](https://vitejs.dev/guide/)

### 4. Environment Variables
The project uses environment variables for database connections and API keys.
* **Action Required:** Copy the provided sample file to create your local environment file:
  ```bash
  cp .env.sample .env

> **Note:** Open the `.env` file and fill in the required credentials before starting the application.

## ⚙️ Configuration

### Environment Files

- `.env.sample`: A template containing all necessary keys (shared in the repository).
- `.env`: Your local, private configuration (ignored by Git).
- `.deploy.json`: Stores server connection details for the deploy task.
- `kozitabor-api/.env.development, kozitabor-react/.env.development`: Variables for the local development environment

> ⚠️ **Important:** Never commit your `.env` or `.deploy.json` files to version control. They are already included in the `.gitignore`.

## 🏗️ DevTool - The Automation Engine

The `devtool.js` located in the root directory provides an interactive CLI for common development and operations tasks.

### Key Features:
1.  **Development Environment:**
    * **Initialization:** Automated installation of required dependencies for both Frontend and Backend.
    * **Local Execution:** Starts the Vite development server with Hot Module Replacement (HMR).
    * **Data Seeding:** Includes a built-in seeding utility to populate the database with initial or test data for a consistent development experience.

2.  **Build Pipeline:**
    * Runs the Vite compiler for the React frontend.
    * Builds Docker images for both the API and the Frontend.
    * Exports images into compressed `.tar.gz` archives.

3.  **Deployment (CI/CD):**
    * **Security:** Supports SSH Key-based authentication (OpenSSH PEM format).
    * **Automated Remote Setup:** Recursively creates the target directory on the server.
    * **Smart Backup:** Automatically creates a timestamped backup of the remote directory before overwriting.
    * **File Transfer:** Synchronizes Docker images and configurations via SCP.
    * **Remote Launch:** Loads images and restarts services using `docker compose up -d`.

---

### Usage

To start the DevTool, simply run:

```bash
node devtool.js
```

Running this command will display the full list of tasks

> **Important:** Database protection is NOT the responsibility of DevTool.  
> The deployment process itself does not inherently put existing data at risk. However, if the deployment encounters issues, a full reset (including container and volumes) may be necessary.  
> **Always ensure you have a complete backup of your database before performing such operations!**  
>  
> Additionally, previous files such as images, `.env`, `docker-compose.yml`, `nginx.conf`, and others can be found later in a backup located in the parent directory of the target folder.

### Example: Initializing the project

```bash
node devtool.js init
```

Note: Certain tasks, like deploy, will launch an interactive prompt to collect or confirm server details.

## 📁 Project Structure

* **`/build`**: Temporary artifacts generated by the DevTool.
* **`/development-db`**: Local instance for a docker PostgreeSQL database
* **`/kozitabor-api`**: Backend service source code.
* **`/kozitabor-react`**: React frontend source code.
* **`docker-compose-deploy.yml`**: Orchestration file for deploying the production environment.
* **`docker-compose.yml`**: Orchestration file for building the production environment.
* **`nginx.conf`**: Reverse proxy configuration for web access.
* **`.deploy.json`**: (Auto-generated) Stores the latest successful deployment settings.
